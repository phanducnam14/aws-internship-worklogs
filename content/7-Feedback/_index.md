## Sharing and Feedback

After completing the internship journey at **AWS First Cloud - AI Journey**, I would like to share some personal reflections and feedback about the program. This internship gave me the opportunity to improve my technical foundation, experience a cloud learning environment, and understand how AWS services can be applied to a real project.

## Overall Evaluation

### 1. Learning and Practice Environment

The internship environment provided a good balance between self-study, hands-on practice, and community learning. Instead of only learning theoretical knowledge, I had the chance to work directly with AWS services and understand how each service is used in real cloud architecture.

The program also created a dynamic and open learning atmosphere where interns could ask questions, discuss problems, and improve through practical exercises. This helped me become more familiar with the working style of the technology field.

### 2. Support from Mentor and Admin Team

The mentor and admin team supported interns throughout the program by providing learning directions, study materials, and guidance during important stages. When I encountered difficulties in understanding AWS services or planning the final project architecture, the guidance from the program helped me identify what I needed to learn next.

The admin team also provided necessary information and reminders, which helped interns follow the internship timeline and complete required reports more smoothly.

### 3. Relevance to My Academic Major

The internship was highly related to my Software Engineering major. Many topics such as backend systems, cloud deployment, database, authentication, API integration, monitoring, and system architecture were connected to what I had learned at university.

Through this internship, I was able to apply academic knowledge to a more practical environment. For example, concepts such as client-server architecture, authentication, database storage, and API communication became clearer when I connected them with AWS services like S3, CloudFront, Cognito, API Gateway, Lambda, DynamoDB, and CloudWatch.

### 4. Learning and Skill Development Opportunities

The program helped me improve both technical and soft skills. Technically, I learned more about AWS core services, cloud architecture, IAM permission management, serverless services, monitoring, and cost control.

In the final project phase, I had the opportunity to support the team by focusing on AWS service setup and integration. This helped me understand that deploying a cloud system requires not only coding knowledge, but also service configuration, permission setup, debugging, and teamwork.

Besides technical skills, I also improved my self-learning ability, documentation skills, and task reporting. I learned how to summarize weekly work, explain what I had done, and connect my learning process with the project outcome.

### 5. Teamwork and Collaboration

Teamwork was an important part of the internship. During the final project, each member had different responsibilities such as frontend, backend, testing, and AWS integration. My role was mainly related to preparing and configuring AWS services for the system.

Through team discussions and study sessions, I learned how to listen to other members, understand the project flow, and adjust my work to match the overall system. This helped me realize that a successful project depends not only on individual effort, but also on clear communication and coordination among team members.

### 6. Internship Policy and Learning Flexibility

The program provided a suitable learning structure for students. The flexible learning format allowed me to balance internship tasks, university requirements, and final project preparation.

The use of AWS practice resources also helped me gain hands-on experience with real cloud services. Because AWS services can generate costs, I also learned the importance of checking billing, setting budgets, and cleaning up unused resources after practice.

## Personal Reflections and Takeaways

### Most Satisfying Aspect During the Internship

The most satisfying part of the internship was being able to move from basic AWS learning to applying AWS services in a real group project. At the beginning, I only learned services separately, such as EC2, S3, IAM, VPC, and Lambda. Later, I gradually understood how these services can be connected to form a complete system.

I also found the AWS Vietnam Community Day event meaningful because it helped me gain more practical insights from real speakers and cloud practitioners. The event gave me a broader view of AWS, AI, serverless architecture, and career development in the cloud field.

### Areas for Program Improvement

For future cohorts, the program could provide more sample architecture diagrams and basic deployment templates during the early weeks. This would help interns understand the relationship between services more quickly and reduce confusion when starting the final project.

It would also be helpful to have more short sharing sessions between different teams. Each team may use different AWS services or face different problems, so sharing experiences could help interns learn from one another and improve their project implementation.

### Advice for Future Interns

Future interns should not only watch videos or read documents, but also practice directly on the AWS Console. AWS services are easier to understand when they are tested in real situations.

They should also pay attention to IAM permissions, service costs, and CloudWatch logs from the beginning. These are common areas where beginners may face problems during deployment.

In addition, interns should take weekly notes carefully. Keeping a clear worklog makes it much easier to write reports, review progress, and prepare for final evaluation.

## Suggestions and Expectations

### Suggestions for a Better Experience

The internship experience could be improved by providing quick-start guides for common AWS services such as S3 static website hosting, CloudFront, API Gateway, Lambda, Cognito, DynamoDB, and CloudWatch. These guides would help interns avoid basic setup mistakes and spend more time understanding architecture and system flow.

It would also be useful to include a short troubleshooting guide for common errors, such as permission denied issues, Lambda runtime errors, API Gateway CORS errors, and CloudFront deployment problems.

### Future Alignment

After this internship, I want to continue improving my knowledge in AWS, backend development, and cloud deployment. I also hope to apply what I learned from the program to future projects, especially in areas related to DevOps, cloud infrastructure, serverless architecture, and backend system deployment.

In the future, I would like to keep learning from the AWS community and participate in more cloud-related activities to improve my technical skills and career readiness.
